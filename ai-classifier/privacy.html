Privacy Policy

Last updated: January 19, 2025

Summary: We collect minimal data necessary to provide the service. Ticket content is sent to Claude API for classification. Classification patterns and feedback are stored in Forge Storage within Atlassian's infrastructure. No data leaves Atlassian except to the AI provider. We don't sell your data.

1. Introduction

AI Ticket Classifier ("we", "our", or "the App") is a Jira Cloud application that automatically classifies new tickets using AI. This Privacy Policy explains how we collect, use, and protect your information.

2. Information We Collect

2.1 Information from Atlassian

When you install and use our App, we access the following through Atlassian's Forge platform:

  • Jira Issue Data: Summary, description, issue type, priority, assignee, and labels of tickets for classification purposes (read and write access)
  • Historical Issue Data: Past tickets in your project are analyzed to learn classification patterns
  • User Context: Basic information about the current user session provided by Atlassian
  • License Information: Your app license status from Atlassian Marketplace

This data is processed within Atlassian's Forge infrastructure.

2.2 Information Stored in Forge Storage

The following information is stored in Atlassian's Forge Storage (server-side, within Atlassian's infrastructure):

  • Classification Patterns: Learned patterns from your historical tickets used for future classifications
  • Feedback Data: Your team's corrections and confirmations of AI classifications
  • Configuration Settings: Classification rules, thresholds, and project preferences
  • Analytics Data: Aggregated classification accuracy metrics and usage statistics
  • API Key: Your Claude API key (encrypted in Forge Storage)

This data is stored within Atlassian's secure infrastructure and is not transmitted to our servers.

2.3 Information Sent to AI Providers

When the App classifies a ticket:

  • The text content (summary and description) of the Jira ticket is sent to Anthropic's Claude API for classification
  • Your API key is used to authenticate these requests
  • Classification patterns may be included in the prompt context to improve accuracy
  • This data is processed according to Anthropic's privacy policy

3. How We Use Information

We use the information collected to:

  • Classify new tickets by issue type, priority, and assignee
  • Learn classification patterns from historical tickets
  • Improve classification accuracy through feedback
  • Provide analytics and accuracy metrics on the Admin Dashboard
  • Verify your license status

4. Data Storage and Security

4.1 Forge Platform

Our App runs entirely on Atlassian's Forge platform. All data processing and storage occurs within Atlassian's secure infrastructure. We do not operate our own servers for data storage.

4.2 Forge Storage

Classification patterns, feedback data, and configuration are stored in Forge Storage:

  • Stored within Atlassian's secure infrastructure
  • Isolated per Jira instance (your data is not shared with other customers)
  • Subject to Atlassian's security and compliance standards
  • Deleted when the App is uninstalled

4.3 Your API Keys

Your AI provider API keys are:

  • Stored encrypted in Forge Storage
  • Never transmitted to our servers
  • Only sent directly to the AI provider for classification requests

4.4 Jira Data

Your Jira issue data:

  • Is accessed when a new ticket is created or when training the classifier
  • Ticket content is sent to the AI provider only for classification purposes
  • Learned patterns (not raw ticket content) are stored in Forge Storage
  • Raw ticket content is not persisted by our App beyond the immediate classification

5. Third-Party Services

Our App integrates with the following third-party services:

5.1 Atlassian

The App is built on Atlassian Forge and accesses Jira Cloud. See Atlassian's Privacy Policy.

5.2 Anthropic (Claude)

Ticket content is sent to Anthropic's Claude API for classification. See Anthropic's Privacy Policy.

6. Data Retention

Data stored in Forge Storage (classification patterns, feedback, configuration) persists until you:

  • Uninstall the App (all Forge Storage data is deleted)
  • Manually reset training data through the Admin Dashboard
  • Request data deletion by contacting us

Raw ticket content is not retained by the App after classification is complete.

7. Your Rights

You have the right to:

  • Access: View what data the App stores via the Admin Dashboard
  • Deletion: Reset training data or uninstall the App to delete all stored data
  • Portability: Your Jira data remains in Jira and is not separately stored by us
  • Uninstall: Remove the App at any time through Jira's app management

8. Children's Privacy

Our App is not intended for use by children under 16. We do not knowingly collect information from children.

9. Changes to This Policy

We may update this Privacy Policy from time to time. We will notify users of significant changes through the App or via Atlassian Marketplace.
